KMP算法

什么是KMP算法

KMP（Knuth-Morris-Pratt）算法是一种经典的字符串匹配算法，用于解决在一个文本串中快速查找模式串出现位置的问题。相比于暴力匹配算法在失配时需要重新比较已经匹配成功的字符，KMP 能够充分利用已有的匹配信息，将字符串匹配的时间复杂度从最坏情况下的 (O(nm)) 优化到 (O(n+m))。

KMP 的核心思想是在模式串中预处理出一个 next（也称失配函数、前缀函数）数组，用来记录模式串每个前缀的最长相同前后缀长度。当匹配过程中发生失配时，不再将模式串简单地向后移动一位，而是利用 next 数组快速找到模式串下一次应该继续匹配的位置，从而避免对已经比较过的字符进行重复比较，实现线性的匹配效率。

例题

P3375 【模板】KMP

题目描述

给出两个字符串 $s_1$ 和 $s_2$，若 $s_1$ 的区间 $[l, r]$ 子串与 $s_2$ 完全相同，则称 $s_2$ 在 $s_1$ 中出现了，其出现位置为 $l$。
现在请你求出 $s_2$ 在 $s_1$ 中所有出现的位置。

定义一个字符串 $s$ 的 border 为 $s$ 的一个非 $s$ 本身的子串 $t$，满足 $t$ 既是 $s$ 的前缀，又是 $s$ 的后缀。
对于 $s_2$，你还需要求出对于其每个前缀 $s’$ 的最长 border $t’$ 的长度。

输入格式

第一行为一个字符串，即为 $s_1$。
第二行为一个字符串，即为 $s_2$。

输出格式

首先输出若干行，每行一个整数，按从小到大的顺序输出 $s_2$ 在 $s_1$ 中出现的位置。
最后一行输出 $|s_2|$ 个整数，第 $i$ 个整数表示 $s_2$ 的长度为 $i$ 的前缀的最长 border 长度。

对于 $s_2$ 长度为 $3$ 的前缀 ABA，字符串 A 既是其后缀也是其前缀，且是最长的，因此最长 border 长度为 $1$。

数据规模与约定

本题采用多测试点捆绑测试，共有 4 个子任务。

  • Subtask 0（30 points）：$|s_1| \leq 15$，$|s_2| \leq 5$。
  • Subtask 1（40 points）：$|s_1| \leq 10^4$，$|s_2| \leq 10^2$。
  • Subtask 2（30 points）：无特殊约定。
  • Subtask 3（0 points）：Hack。

对于全部的测试点，保证 $1 \leq |s_1|,|s_2| \leq 10^6$，$s_1, s_2$ 中均只含大写英文字母。

P4391 [BalticOI 2009] Radio Transmission 无线传输

题目描述

一家无线电台需要向多位接收者发送一条信息。为了确保所有听众都能接收到，该信息在一个连续的循环中被一遍又一遍地播放。

你将得到其中一位接收者收到的一段字符序列。已知该序列的长度至少与原信息的长度一样长。

你的任务是编写一个程序，提取出电台发送的原信息。更形式化地说，你的程序需要找到输入序列 $S$ 的最短子序列 $S^{\prime}$，使得 $S$ 本身又是（足够长的）重复序列 $S^{\prime}+S^{\prime}+\cdot\cdot\cdot+S^{\prime}$ 的子序列。

输入格式

第一行包含一个整数 $L$，即序列 $S$ 的长度。

第二行包含恰好 $L$ 个字符，即序列 $S$ 本身。该序列由小写字母组成。

输出格式

程序应向标准输出写入一行，包含一个整数：信息 $S^{\prime}$ 的长度 $L^{\prime}$。请注意，$L^{\prime}$ 必须是尽可能小的值。

输出 #1

样例输入输出 1 解释

对于样例，我们可以利用 $\texttt{abc}$ 不断自我连接得到 $\texttt{abcabcabcabc}$，读入的 $\texttt{cabcabca}$，是它的子串。

规模与约定

对于全部的测试点，保证 $1\le L \le 10^6$。

解题思路

这道题乍一看完全不知道怎么做，方法非常巧妙，因为S是若干S’相加的一个字串，所以S肯定存在一个比S’长度要长的最长相同前后缀，用S.length-next[n-1]就是答案。
